本地仓库操作----------
正常阶段

绑定本地仓库的帐号邮箱
git config --global user.name xxx
git config --global user.email xxx

创建本地仓库
mkdir xxx   cd xxx

让git管理此仓库
git init


功能操作指令
git status 查看当前状态
git add xxx1 xxx2 xxx3 将文件存入暂存区  git add. 添加当前目录到暂存区
git commit -m “注释内容”


版本回退阶段
1：查看版本，确定需要回到什么版本
git log  显示操作日志（commitID、操作人、时间、备注）最上面的最新
git log --pretty=oneline 显示操作日志（commitID、备注）
2：回退操作
git reset --hard commitID
3：回退完成之后前进到其他版本
git reflog 查看操作日志 拿到commitID 再次使用git reset --hard commitID 去到指定版本
本地仓库操作----------

操作远程仓库----------
git clone origin 克隆远程仓库到本地
git pull 拉取远程仓库代码
git push 提交代码到远程仓库
操作远程仓库----------

分支
操作git的时间轴就是一个分支，称为master分支
git branch 查看分支 当前分支最前面有个标志
git branch xxx 创建分支
git checkout xxx 切换分支
git branch -d xxx 删除分支

合并分支
1修改或新增先添加到本地版本库
2切换到master主分支
3git merge xxx 合并到master分支

解决冲突
1先git pull 自动合并冲突到冲突文件中
2保留需要的代码
3重新提交到本地仓库再到远程仓库

忽略上传文件
1在根目录新建.gitignore  命令行touch .gitignore
过滤整个文件夹/目录名/ 
过滤所有.xxx后缀文件 *.xxx
过滤具体文件 /mtk/do.c
